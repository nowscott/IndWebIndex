# 更新日志

本文档根据仓库历史提交与版本标签整理。项目当前不再使用自动 GitHub Release workflow，后续版本记录以 `package.json` 版本和人工维护的更新日志为准。

## 未来版本规划

这些方向用于记录后续可能进入版本更新的事项。未完成项使用方框标记；完成后从这里移入对应版本记录。

### 数据与可靠性

- [ ] 增加静态 JSON 导出或缓存快照兜底，避免 Notion 临时故障阻塞新部署。
- [ ] 增加可选的死链检测、重复网址检测和断开连接站点报告。
- [ ] 持续观察标签质量，包括未使用标签、近似重复标签和过于宽泛的分类。
- [ ] 考虑增加手动 revalidate 接口，或由 Notion 数据变更触发更新，让数据库编辑后更快发布到站点。

### 搜索与性能

- [ ] 在 Notion 数据格式化阶段预生成搜索字段，包括小写文本、全拼和首字母。
- [ ] 如果数据量增长到当前页面 payload 难以轻松承载，再引入轻量客户端搜索索引。
- [ ] 随数据增长持续关注首页 HTML 和 `__NEXT_DATA__` 体积；只有当 payload 真正影响用户体验时，再拆分数据加载。

### 用户体验

- [ ] 在当前右键字体菜单之外，增加一个可见的字体菜单入口。
- [ ] 优化搜索、标签筛选和主题切换的键盘操作与屏幕阅读器支持。
- [ ] 为无数据、Notion 配置错误、访问统计未启用等状态提供更清晰的提示。
- [ ] 考虑增加可选的浏览视图，例如最近添加、常用网站或按标签聚合浏览。

## [3.3.3] - 2026-07-01

### 维护

- 删除自动更新 GitHub Release 的 workflow，避免合并后自动 bump 和发版。
- 新增 CI workflow，覆盖 `npm ci`、生产依赖审计、单元测试和生产构建。
- 将项目 Node.js 运行版本统一到 `24.x`。
- 移除未使用的 MongoDB 依赖和遗留连接模块。
- 修复 `form-data@4.0.5` 生产依赖审计漏洞，锁文件更新到安全版本。
- 新增 `.env.example`，明确 Notion 和 Turso 环境变量。
- 增加 `SKIP_NOTION_FETCH=1` 构建开关，方便 CI 在没有私有 Notion 配置时完成构建验证。

### 文档

- 新增本文档，用于补齐版本历史和后续更新计划。
- README 增加本地维护命令、环境变量示例说明和版本历史入口。
- README 将滞后的 `v3.0.0` 描述改为更稳定的 `3.x` 表述。

## [3.0.x] - 2026-03-16 至 2026-03-18

### 新增

- 完成 3.x 前端重构，引入首页统计、页脚版本信息、关于页和全局状态管理。
- 增加动态自适应主题 favicon。
- 支持中文站点名称和标签的拼音搜索。
- 增加动态纸张纹理背景、深色模式背景随机化和彩虹渐变效果。

### 优化

- 多轮调整浅色和深色主题配色、组件边框、背景样式、间距和交互动效。
- 优化字体菜单、字体预加载和主题切换动画性能。
- 使用 `useMemo` 优化主页面过滤性能。
- 升级 Next.js，并补充 Node.js 引擎约束。

### 修复

- 修复标签筛选和搜索逻辑错误。
- 修正统计区显示的网页数量为可见网页数量。
- 修复 React 属性大小写警告、搜索框占位符、重复 favicon 和移动端主题色问题。
- 修复 Next.js 实验性配置兼容问题。

### 文档

- 多次更新 README、关于页和站点文案，明确项目定位和品牌表达。

## [2.x] - 2024-05-21 至 2026-03-16

- 引入 Next.js/Vercel 部署链路，并多次调整构建、部署和缓存配置。
- 增加访问次数统计尝试，早期曾使用 MongoDB 方案。
- 增加主题切换、字体切换、右键字体菜单、搜索、标签筛选和样式优化。
- 持续更新站点数据、图标、README 和部署说明。
